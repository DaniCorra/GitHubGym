Branching

Il modo corretto per lavorare su github è quello di fare intenso uso del branching, una sorta di "copia privata" di lavoro

Per farlo:

  1. Fate click (nella sezione Code) sul pulsante [Branch: master]
  2. Scrivete il nome della vostra branch - esempio "[vostro nome]-modifiche"
  3. Ora state lavorando sul vostro branch. Potete passare liberamente tra l'uno e l'altro con il menu
  4. Fate le modifiche come nel caso precedente, e committate. Notata che solo il vostro branch è cambiato
  5. Ora è il momento di segnalarlo a chi di dovere tramite una pull request
  6. Andate sulla linguette [Pull request] e poi click sul pulsante [New Pull Request]
  7. Scegliete due branch separati (di solito: il vostro e il master) e vedete se è possibile farlo automaticamente
  8. Commentate, fate una segnalazione @[un altro utente], fate modifiche . In futuro le legheremo alle issues
  9. Dopo la discussione o un via libera, si può tentare il merge. Scegliete una pull request dalla lista
  10. Se il merge si può fare in automatico, l'operazione è istantanea
  11. Altrimenti, occorre "spulciare" il codice riga x riga, scegliendo quale delle due versioni tenere
  12. Fatto questo, clickare su "Marked as resolved"
  13. Eliminati tutti i conflitti, cliccare su "Marked as resolved" e Mergiare come sopra. Potete lavorare anche sulle pull request degli altri....è un mondo collaborativo!
